\section*{Решаемая в статье проблема}

Наиболее общая проблема:

\begin{itemize}
  \item Выразительность текстовых языков в принципе ограничена, поскольку они представляют описываемую систему в последовательной форме.
  \begin{itemize}
    \item Текстовые языки не позволяют одновременно рассматривать систему с нескольких различных точек зрения.
    \item Текстовые языки слабо подходят для описания принципиально непоследовательных систем, систем со сложной структурой и систем
      со сложными взаимосвязями между компонентами.
    \item Текстовые языки не всегда интуитивно понятны не знакомому с ними человеку.
  \end{itemize}
\end{itemize}

Одним из путей решения этой проблемы являются визуальные языки. Специфика разработки аппаратного обеспечения:

\begin{itemize}
  \item Для аппаратного обеспечения характерна компонентная структура и большое количество достаточно сложных связей между компонентами.
  \item Аппаратное обеспечение принципиально паралельно.
  \item К аппаратному обеспечению обычно есть жёсткие требования, касающиеся быстродействия.
  \item Функциональность может быть достаточно произвольным образом разделена между программными и аппаратными компонентами.
  \item Необходимость организовать тесное взаимодействие между аппаратным и программным обеспечением.
  \begin{itemize}
    \item Проблема коммуникации между специалистами из области аппаратного и программного обеспечения.
  \end{itemize}
\end{itemize}

Узкая специфика технологий визуального программирования (в отличие от моделирования, где не требуется исполнимость и полнота моделей):
\begin{itemize}
  \item По визуальной модели сложно сразу сгенерировать исполняемый код на низкоуровневом языке.
  \item Диаграммы, полностью специфицирующие систему, получаются слишком громоздкими и сложными для восприятия.
\end{itemize}
Применение промежуточного текстового языка для решения указанных выше проблем приводит к возникновению новой проблемы: организации взаимодействия
графического языка и текстового языка, который будет использоваться для генерации. Это делает неэффективным применение универсальных визуальных языков.
  
Конкретные проблемы языка системы CoolKit, которые можно решить путём введения визуального языка:

\begin{itemize}
  \item Текстовый язык недостаточно наглядно выражает отношения между процессами.
  \begin{itemize}
    \item Отображения портов.
    \item Объявление и применение функторов.
    \item Отношения использования процессов.
  \end{itemize}
  Эта проблема является общей для всех текстовых языков, различается лишь используемая терминология.
  \item Процесс может использовать другие процессы тремя способами: определять внутри себя как вложенный процесс, использовать процесс извне как вложенный,
    или использовать процесс как параметр. Если система достаточно большая, текстовое представление структуры использования процессов может быть весьма
    сложным. 

    Конкретные формы использования специфичны для языка CoolKit (что делает неэффективным применение существующих визуальных языков), однако, любой 
    текстовый язык представляет подобные сложные структуры недостаточно наглядно.
\end{itemize}
